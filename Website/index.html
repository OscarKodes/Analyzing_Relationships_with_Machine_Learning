What can we learn about relationships by using machine learning for exploratory analysis?

The dataset was taken from a Stanford University survey called, "How Couples Meet and Stay Together 2017." This dataset contains many variables including age, education level, political party, income, usage of dating apps, and quality of relationship (as self-rated by the subject). To prepare this data for analysis, I renamed and recoded a few features. The specific cleaning steps I took can be seen in this Jupyter notebook. During the cleaning process, I also consulted this codebook.

For this project, I had a few general research questions:
  • Can any features in this dataset help predict a subject's relationship quality rating?
  • Do couples that meet on dating apps have higher or lower quality relationships?
  • What insights can I derive from using machine learning for exploratory analysis?

In Principal Component Analysis (PCA), the data is represented by principal components. PCA searches for the areas in the dataset that have the most variation and creates an "axis" for it. This axis with high variance is called a pricipal component. PCA is a great way to reduce the number of dimensions in a dataset without losing too much information.

PCA also sorts its components from most to least variance captured, so a decent way to visualize a high-diemensional dataset on a 2D plane is by using the first two components as X and Y axes.

For my PCA analysis, I took only the top features from the final classificatoin and regression models to make interpretation simpler.

PCA components as axes on a plot can be difficult to interpret because they take in information from many different features. Visualizing the components on a heatmap first might help interpretation.

Feature Contributions to Components 1 & 0

The color of each feature shows how it contributes to that component's variance and how they relate to one another for that specific component. This can be used to help interpret a plot where the axes are components 0 and 1.

The grey colored rectangles show that a feature does not contribute much to a component's variance.

A yellow colored feature shows that it is positive, so when you increase on this component's axis, you will get more of that feature.

For example, Component 0 has the number of household minors colored in yellow, so as you go forward on this axis, the subjects tend to have more children at home.

A black colored feature shows that it is negative, so when you increase on this component's axis, you will get less of that feature.

For example, Component 0 has the partner's age colored in black, so as you go forward on this axis, the cases tend to have younger partners.
